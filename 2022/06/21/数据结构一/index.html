时间复杂度

算法时间复杂度以算法中基本操作重复执行的次数（简称为频度）作为算法的时间度量。

\(O(1)\)<\(O(log_2n)\)<\(O(n)\)<\(O(nlog_2n)\)<\(O(n^2)\)<\(O(n^3)\)<\(O(2^n)\)<\(O(n!)\)<\(O(n^n)\)

常数阶<对数阶<线性阶<线性对数阶<平方阶<立方阶<指数阶<阶乘阶<N次方阶

加法规则：多项相加，保留最高阶项，并将系数化为1

乘法规则：多项相乘都保留，并将系数化为1

加法乘法混合规则：小括号再乘法规则最后加法规则

空间复杂度

\(O(1)\),\(O(n)\),\(O(n^2)\)

渐进符号

  1. O记号，渐进上界
  2. Ω记号，渐进下界
  3. Θ记号，渐进紧致界

递归式 时间复杂度，空间复杂度

（每次递归的时间复杂度不变的情况）时间复杂度=递归的次数 * 每次递归的时间复杂度

主方法，主定理，求解递归

线性结构

是一种基本数据结构，对单一前驱和后继的数据关系进行描述（即元素一个接一个排列）

线性表

常采用顺序存储和链式存储，基本操作是插入，删除和查找

定义：一个线性表是n(\(n\ge0\))个元素的有限序列，通常表示为（a1,a2,a3...an）.

线性表存储结构

  1. 顺序存储

     用一组地址连续的存储单元依次存储线性表中的数据元素。从而使逻辑上相邻的两个元素物理位置上也相邻。
 优点：随机存取表中的元素
 缺点：插入和删除操作需要移动元素
 插入的时间复杂度：最好O(1),最坏O(n),平均O(n)
 删除的时间复杂度：最好O(1),最坏O(n),平均O(n)
 查找的时间复杂度：最好O(1),最坏O(1),平均O(1)

  2. 链式存储

     用通过指针链接起来的结点来存储数据元素（数据域｜指针域）
 数据域存储数据元素值，指针域存储当前元素的直接前驱或直接后继的位置信息
 带头结点
         插入时间复杂度：最好O(1),最坏O(n),平均O(n)
         删除时间复杂度：最好O(1),最坏O(n),平均O(n)
         查找的时间复杂度：最好O(1),最坏O(n),平均O(n)
 不带头结点
         插入时间复杂度：最好O(1),最坏O(n),平均O(n)
         删除时间复杂度：最好O(1),最坏O(n),平均O(n)
         查找的时间复杂度：最好O(1),最坏O(n),平均O(n)

栈

栈是只能通过访问它的一端来实现数据存储和检索的一种线性数据结构。

栈也被称为是后进先出（LIFO,Last In First Out）的线性表

在栈中进行插入和删除操作的一端称为栈顶（Top）,另一端称为栈底（Bottom）

队列

队列是一种先进先出（FIFO,First in First Out）的线性表,只允许在表的一端插入元素，在表的另一端删除元素。在队列中，允许插入元素的一端称为队尾（Rear）,允许删除元素的一端称为队头（Front）

队列的顺序存储

队列的顺序存储结构又称为顺序队列，它是利用一组地址连续的存储单元存放队列中的元素，设置队头指针和队尾指针。

队列的链式存储

串

串的模式匹配

    n:主串，m:子串
    最好：O(m)
    最坏：O(n*m)
    平均：O(n+m)

KMP

    串的前缀：包含第一个字符并且不包含最后一个字符的子串
    串的后缀：包含最后一个字符并且不包含第一个字符的子串
    第i个字符的next值=从1～i-1串中最长相等前后缀长度+1
    特殊情况：next[1]=0

一维数组

LOC：第一个元素的首地址，L：元素大小

下标从0开始：ai=LOC+i*L

下标从1开始：ai=LOC+(i-1)*L ### 二维数组 LOC：第一个元素的首地址，N:行数，M:列数，L：元素大小

按行优先存储并且下标从0开始：ai,j=LOC+(iM+j)L

按行优先存储并且下标从1开始：ai,j=LOC+[(i-1)*M+(j-1)]*L

按列优先存储并且下标从0开始：ai,j=LOC+(jN+i)L

按列优先存储并且下标从1开始：ai,j=LOC+[(j-1)*N+(i-1)]*L

当i>=j,Aij=i(i+1)/2+j+1

当i<=j,Aij=j(j+1)/2+i+1

稀疏矩阵

树

非线性结构,一对多（0个或多个）

树的定义

树是n(n>=0)个结点的有限集合，当n=0时称为空树，在任一非空树(n>0)中，有且仅有一个称为根的结点。

树的基本概念

  1. 双亲，孩子和兄弟
  2. 结点的度：一个结点的字树的个数记为该结点的度
  3. 叶子结点：也称为终端结点，指度为0的结点
  4. 分支（内部）结点：度不为0的结点
  5. 结点的层次：根为第一层，根的孩子为第二层......
  6. 树的高度：一棵树的最大层数记为树的高度（或深度）

树的性质

  1. 树中的结点总数等于树中所有结点的度数之和加1
  2. 度为m的树中第i层上至多有m^i-1个结点（i>=1）
  3. 高度为h的m次树至多有(m^h -1)/(m-1)个结点
  4. 具有n个结点，度为m的树的最小高度为logm(n(m-1)+1)

二叉树

定义：二叉树是n(n>=0)个结点的有限集合，或者是空树（n=0）,或者是由一个根结点及两颗不相交的且分别称为左，右子树的二叉树所组成,二叉树结点最大度为2

二叉树的性质

  1. 二叉树第i层（i>=1）上最多有2^i-1个结点
  2. 高度为h的二叉树最多有2^h -1个结点
  3. 对于任何一棵二叉树，度为0的结点数等于度为2的结点数+1
  4. 具有n个结点的完全二叉树的高度为log2n +1或者 log2(n+1)

满二叉树与完全二叉树

高度为k的二叉树有2^k -1 个结点，则称其为满二叉树

高度为h的完全二叉树中，除h层其余各层都满，第h层从左至右放置

卡特蓝数

二叉树的存储结构

  1. 二叉树的顺序存储结构，顺序存储是用一组地址连续的存储单元存储二叉树中的结点，完全二叉树适合顺序存储结构
  2. 二叉树的链式存储结构，二叉链表：n个结点有2n个指针域，n-1个有效指针域，n+1个空指针域，三叉链表：n个结点有3n个指针域，n+2个空指针域

二叉树的遍历算法

  1. 先序遍历 根左右
  2. 中序遍历 左根右
  3. 后序遍历 左右根
  4. 层次遍历

平衡二叉树

二叉树中的任意一个结点的左右子树高度之差的绝对值不超过1

二叉排序树（二叉查找树）

根结点的关键字 大于左子树所有结点的关键字，小于右字数所有结点的关键字，左右子树也是一颗二叉排序树

中序遍历得到的序列是有序数列

最优二叉树（哈夫曼树）

它是一类带权路径长度最短的树。结点的带权路径长度为从该结点到树根之间的路径长度与该结点权值的乘积 \[WPL=\sum_{k=1}^nw_kl_k\] \(n\)为带权叶子结点数目，\(w_k\)为叶子结点的权值，\(l_k\)为叶子结点到根的路径长度

如何构造最优二叉树

  1. 从前往后找两个权值最小
  2. 小左大右
  3. 加入末尾
  4. 权值相同 从前往后
  5. 用时再调

最优二叉树不唯一，但其WPL是唯一确定的

图

图的定义

图G是由集合V和E构成的二元组，记作\(G=(V,\,E)\),其中，V是图中顶点的非空有限集合，E是图中边的有限集合。

  1. 有向图 \(<V_i,V_j>\)
  2. 无向图 \((V_i,V_j)\)
  3. 完全图 无向完全图，有向完全图
  4. 度，出度，入度 顶点V的度是指关联于该顶点的边的数目，记作\(D(v)\)。若G为有向图，顶点的度表示该顶点的入度和出度之和，记作\(ID(v)\),\(OD(v)\)。无论是有向图还是无向图，顶点数n,边数e与各顶点的度之间有以下关系: \[e=\frac{1}{2}\sum_{i=1}^nD(v_i)\]
  5. 路径
  6. 连通图与连通分量。在无向图G中，若从顶点\(v_i\)到顶点\(v_j\)有路径，则称顶点\(v_i\)和顶点\(v_j\)是连通的。如果无向图G的任意两个顶点都是连通的，则称其为连通图。无向图G的极大连通子图称为G的连通分量。
  7. 强连通图与强连通分量。在有向图G中，如果对于每一对顶点\(v_i,v_j \in V\)且\(v_i \neq v_j\)，从顶点\(v_i\)到顶点\(v_j\)和从顶点\(v_j\)到顶点\(v_i\)都存在路径，则称图G为强连通图。有向图中的极大连通子图称为有向图的强连通分量。

图的存储结构

  1. 邻接矩阵表示法 用矩阵来表示顶点之间的关系。
  2. 邻接链表表示法 为图的每个顶点建立一个链表

稠密图（边多，用邻接矩阵），稀疏图（边少，用邻接表）

网

边（或弧）带权值的图称为网。

图的遍历

是指从某个顶点出发，沿着某条搜索路径对图中的所有顶点进行访问且只访问一次的过程。

深度优先遍历（Depth First Sesrch,DFS）

实现思想：递归

n为顶点数，e为边数

当图用邻接矩阵表示时，查找所有顶点的邻接点所需时间为\(O(n^2)\)

当图用邻接表表示时，深度优先搜索遍历图的时间复杂度为\(O(n+e)\)

广度优先遍历（Breadth First Sesrch,BFS）

每个顶点最多进一次队列

运算时间复杂度和深度一致
