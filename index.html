依赖(一条可能有方向的虚线)

依赖是两个事物间的语义关系，其中一个事物（独立事物）发生变化会影响另一个事物（依赖事物）的语义。一条可能有方向的虚线

关联

是一种结构关系，它描述了一组链，链是对象之间的连接。 * 聚合 * 组合

聚合(实线空心菱形，指向整体)

部分和整体的生命周期不一致，整体消失了，部分仍然存在，部分可以脱离整体存在

组合(实线实心菱形，指向整体)

部分和整体的生命周期一致，整体消失了，部分也消失，部分不可以脱离整体存在

关联多重度

  • 单项关联(一条有方向的实线)

泛化(实线空心箭头，指向父类)

泛化是一种特殊/一般关系，特殊元素(子元素)的对象可替代为一般元素(父元素)的对象。用这种方法，子元素共享父元素的结构和行为。

实现(虚线空心箭头，指向父类)

一种是在接口和实现类之间，另一种是在用例和实现它们的协作之间

包过滤防火墙

  • 低水平控制 ：每个IP包的字段都被检查（源地址，目的地址，协议和端口）
  • 对用户完全透明，速度较快

应用代理网关防火墙

用代理网关防火墙彻底隔断内网与外网的直接通信，内网用户对外网的访问变成防火墙对外网的访问，然后再由防火墙转发给内网用户。 * 优点是可以检查应用层，传输层和网络层的协议特征，对数据包的检测能力比较强 * 缺点是难以配置，处理速度非常慢

关系模型中的基本术语

  • 关系：一个关系就是一张二维表，每个关系有一个关系名。
  • 元组：表中的一行即为一个元组，对应存储文件中的一个记录值。
  • 属性：表中的列称为属性。
  • 域：属性的取值范围。
  • 关系模式：对关系的描述称为关系模式，由关系名和其属性集合构成，关系模式的格式为：
  • 候选码（或候选键）：属性或属性组合，其值能够唯一地标识一个元组。
  • 主码（或主键）：在一个关系中可能有多个候选码，从中选择一个作为主码。
  • 主属性：包含在任何候选码中诸属性称为主属性。
  • 外码（或外键）：如果一个关系中的属性或属性组并非该关系的码，但它是另外一个关系的码，则称其为该关系的外码。
  • 全码：所有属性组是这个关系模式的候选码，称为全码。
  • 超码（超键）：一个包含码的属性集称为超码。

关系模型

关系模型由关系数据结构，关系操作集合和关系完整性约束三部分组成。 1. 关系数据结构 关系模型中数据的逻辑关系是一张二维表格。 2. 关系操作集合（关系代数） 关系模型给出了关系操作的能力。常用的关系操作有：选择，投影，连接，除，并，交，差等查询操作和增加，删除操作等。 3. 关系完整性约束 * 实体完整性：关系中主码的值不能为空或部分为空。 * 参照完整性：外键必须对应上。 * 用户定义的完整性：指用户对某一具体数据指定的约束条件进行检验。

  • 第一范式(1NF) :不可分割的数据项，不能排除数据冗余和更新异常等问题，因为其中可能存在部分函数依赖。

查找

查找是一种常见的基本运算。查找表是指由同一类型的数据元素（或记录）构成的集合。查找表是一种非常灵活的数据结构。

静态查找表有：顺序查找，折半（二分）查找，分块查找 1. 查询某个特定元素是否在查找表中 2. 检索某个特定的数据元素的各个属性

动态查找表有：二叉排序树，平衡二叉树，B_树，哈希表 1. 在查找表中插入一个数据元素 2. 在查找表中删除一个数据元素

关键字是数据元素（或记录）的某个数据项的值，用它来识别（标识）这个数据元素。主关键字是指能唯一标识一个数据元素的关键字。次关键字是指能标志多个数据元素的关键字。

顺序查找

顺序查找的方法对于顺序存储方式和链式存储方式的查找表都适用。

顺序查找成功的平均查找长度为 \[ASL_{ss}=\sum_{i=1}^nP_iC_i=\frac{1}{n}\sum_{i=1}^n(n-i+1)=\frac{n+1}{2}\]

顺序存储，有序

最多的比较次数为 \[\lfloor\log_2n\rfloor+1\] 折半查找成功的平均查找长度为 \[ASL_{bs}=\sum_{j=1}^nP_iC_i=\frac{1}{n}\sum_{j=1}^nj\times2^{j-1}=\frac{n+1}{n}\log_2{(n+1)}-1\] 当n值较大时，\(ASL_{bs}\approx\log_2{(n+1)}-1\) # 哈希表

非线性结构,一对多（0个或多个）

树的定义

树是n(n>=0)个结点的有限集合，当n=0时称为空树，在任一非空树(n>0)中，有且仅有一个称为根的结点。

树的基本概念

  1. 双亲，孩子和兄弟
  2. 结点的度：一个结点的字树的个数记为该结点的度
  3. 叶子结点：也称为终端结点，指度为0的结点
  4. 分支（内部）结点：度不为0的结点
  5. 结点的层次：根为第一层，根的孩子为第二层......
  6. 树的高度：一棵树的最大层数记为树的高度（或深度）

树的性质

  1. 树中的结点总数等于树中所有结点的度数之和加1
  2. 度为m的树中第i层上至多有m^i-1个结点（i>=1）
  3. 高度为h的m次树至多有(m^h -1)/(m-1)个结点
  4. 具有n个结点，度为m的树的最小高度为logm(n(m-1)+1)

二叉树

定义：二叉树是n(n>=0)个结点的有限集合，或者是空树（n=0）,或者是由一个根结点及两颗不相交的且分别称为左，右子树的二叉树所组成,二叉树结点最大度为2

二叉树的性质

  1. 二叉树第i层（i>=1）上最多有2^i-1个结点
  2. 高度为h的二叉树最多有2^h -1个结点
  3. 对于任何一棵二叉树，度为0的结点数等于度为2的结点数+1
  4. 具有n个结点的完全二叉树的高度为log2n +1或者 log2(n+1)

满二叉树与完全二叉树

高度为k的二叉树有2^k -1 个结点，则称其为满二叉树

高度为h的完全二叉树中，除h层其余各层都满，第h层从左至右放置

卡特蓝数

  2. 二叉树的链式存储结构，二叉链表：n个结点有2n个指针域，n-1个有效指针域，n+1个空指针域，三叉链表：n个结点有3n个指针域，n+2个空指针域

平衡二叉树

二叉树中的任意一个结点的左右子树高度之差的绝对值不超过1

二叉排序树（二叉查找树）

根结点的关键字 大于左子树所有结点的关键字，小于右字数所有结点的关键字，左右子树也是一颗二叉排序树

中序遍历得到的序列是有序数列

最优二叉树（哈夫曼树）

它是一类带权路径长度最短的树。结点的带权路径长度为从该结点到树根之间的路径长度与该结点权值的乘积 \[WPL=\sum_{k=1}^nw_kl_k\] \(n\)为带权叶子结点数目，\(w_k\)为叶子结点的权值，\(l_k\)为叶子结点到根的路径长度

如何构造最优二叉树

  1. 从前往后找两个权值最小
  2. 小左大右
  3. 加入末尾
  4. 权值相同 从前往后
  5. 用时再调

最优二叉树不唯一，但其WPL是唯一确定的

图的定义

图G是由集合V和E构成的二元组，记作\(G=(V,\,E)\),其中，V是图中顶点的非空有限集合，E是图中边的有限集合。

  1. 有向图 \(<V_i,V_j>\)
  2. 无向图 \((V_i,V_j)\)
  3. 完全图 无向完全图，有向完全图
  4. 度，出度，入度 顶点V的度是指关联于该顶点的边的数目，记作\(D(v)\)。若G为有向图，顶点的度表示该顶点的入度和出度之和，记作\(ID(v)\),\(OD(v)\)。无论是有向图还是无向图，顶点数n,边数e与各顶点的度之间有以下关系: \[e=\frac{1}{2}\sum_{i=1}^nD(v_i)\]
  5. 路径
  6. 连通图与连通分量。在无向图G中，若从顶点\(v_i\)到顶点\(v_j\)有路径，则称顶点\(v_i\)和顶点\(v_j\)是连通的。如果无向图G的任意两个顶点都是连通的，则称其为连通图。无向图G的极大连通子图称为G的连通分量。
  7. 强连通图与强连通分量。在有向图G中，如果对于每一对顶点\(v_i,v_j \in V\)且\(v_i \neq v_j\)，从顶点\(v_i\)到顶点\(v_j\)和从顶点\(v_j\)到顶点\(v_i\)都存在路径，则称图G为强连通图。有向图中的极大连通子图称为有向图的强连通分量。

图的存储结构

  1. 邻接矩阵表示法 用矩阵来表示顶点之间的关系。
  2. 邻接链表表示法 为图的每个顶点建立一个链表

稠密图（边多，用邻接矩阵），稀疏图（边少，用邻接表）

网

边（或弧）带权值的图称为网。

图的遍历

是指从某个顶点出发，沿着某条搜索路径对图中的所有顶点进行访问且只访问一次的过程。

深度优先遍历（Depth First Sesrch,DFS）

实现思想：递归

n为顶点数，e为边数

当图用邻接矩阵表示时，查找所有顶点的邻接点所需时间为\(O(n^2)\)

当图用邻接表表示时，深度优先搜索遍历图的时间复杂度为\(O(n+e)\)

广度优先遍历（Breadth First Sesrch,BFS）

每个顶点最多进一次队列

运算时间复杂度和深度一致
