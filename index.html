计算机系统基础知识

计算机系统硬件基本组成
控制器，运算器，存储器，输入设备，输出设备

CPU

运算器，控制器被集成在CPU(Central Processing Unit,CPU,中央处理单元)
CPU功能

程序控制，操作控制，时间控制，数据处理

运算器组成

  1. 算术逻辑单元（ALU）arithmetic and logic unit
  2. 累加寄存器（AC）accumulation register
  3. 数据缓冲寄存器（DR）Data register
  4. 状态条件寄存器（PSW）program status word

控制器组成

  1. 指令寄存器（IR）Instruction Register
  2. 程序计数器（PC）Program Counter
  3. 地址寄存器（AR）Address Register
  4. 指令译码器（ID）Instruction Decoder

指令：操作码+地址码

计算机基本单位

名称 EN 简称 换算

字节 byte B 1B=8b
千字节 KB 1KB=1024B
兆字节 MB 1MB=1024KB

太字节 TB 1TB=1024GB

最小的数据单位：b

最小的存储单位：byte

进制转换

名称 简称 EN 一位表示范围
十进制 D Decimal System 0～9
二进制 B Binary number 0～1
八进制 O Octal 0～7
十六进制 H Hex 0～15 （0～9 a～f）

方法

  1. 按权展开求和 n进制->十进制
  2. 除n取余法 十进制->n进制

一位十六进制的数=4位二进制的数
一位八进制的数=3位二进制的数

进制加减法

加法 逢n进1

减法 逢n借1

数据表示

原码表示法：最高位为符号位，0表示正号，1表示负号，其余的n-1表示数值的绝对值。

0的原码有两种形式：[+0]原=0 0000000，[-0]原=1 0000000

反码表示法：最高位为符号位，0表示正号，1表示负号，正数的反码与原码相同，负数的反码则是其绝对值按位求反。

0的反码有两种形式：[+0]反=0 0000000，[-0]反=1 1111111

补码表示法：最高位为符号位，0表示正号，1表示负号，正数的补码与原码，反码相同，负数的补码等于其反码的末尾加1

0有唯一的编码：[+0]补=0 000000，[-0]补=0 000000

移码表示法：补码的符号取反

补码的补码等于原码

码制 定点整数 定点小数
原码 -(2^n-1 -1) ~ +(2^n-1 -1) -(1-2^-(n-1))~+(1-2^-(n-1))
反码 -(2^n-1 -1) ~ +(2^n-1 -1) -(1-2^-(n-1))~+(1-2^-(n-1))
补码 -2^n-1 ~ +(2^n-1 -1) -1 ~ +(1-2^-(n-1))
移码 -2^n-1 ~ +(2^n-1 -1) -1 ~ +(1-2^-(n-1))

寻址

立即寻址：操作数就在指令中。

直接寻址：操作数存放在内存单元中，指令中直接给出操作数所在存储单元的地址。

寄存器寻址：操作数存放在某一寄存器中，指令中给出存放操作数的寄存器名。

寄存器间接寻址：操作数存放在内存单元中，操作数所在存储单元的地址在某个寄存器中。

间接寻址：指令中给出操作数地址的地址。

校验码

码距=2，有检错能力
码距>=3，才可能有纠错能力

  1. 奇偶检验码
  2. 海明码（设数据位是n位，校验位是k位，则n和k必须满足以下关系：
    2^k -1 >= n+k）（利用多组数位的奇偶性来检错和纠错）
  3. 循环冗余校验码(可以检错，不能纠错，码距=2)

指令集

精简指令集RISC 复杂指令集CISC
指令种类 少，精简 多，丰富
指令复杂度 低（简单） 高（复杂）
指令长度 固定 变化
寻址方式 少 复杂多样
实现（译码）方式 硬布线控制逻辑（组合逻辑控制器） 微程序控制技术
通用寄存器数量 多，大量 一般
流水线技术 支持 不支持

指令流水线

流水线:第1条指令的执行时间+（n-1）*(最长的时间段)

加速比=不采用流水线/采用流水线

操作周期=最长时间段

吞吐率=1/最长时间段

执行n条指令的吞吐率=n/流水线

存储器

按存储器所处位置可分为：内存，外存

按存储器工作方式可分为：

  1. 读写存储器RAM
  2. 只读存储器ROM
  3. 闪速存储器Flash Memory,可以整块删除

按访问方式可分为：地址访问存储器，内容访问存储器

按寻址方式可分为：

  1. 随机存储器 RAM
  2. 顺序存储器 SAM
  3. 直接存储器 DAM

相联存储器是一种按内容访问的存储器

虚拟存储器由主存，辅存构成

DRAM 动态随机存储，是一种需要通过周期性刷新来保持数据的存储器件

Cache

高速缓存用于存放最活跃的程序和数据，特点：位于CPU与主存之间；容量一般在几千字节到几兆字节之间

高速缓存对程序员来说是透明的

Cache与主存地址的映射是由硬件自动完成

替换算法

替换算法的目标就是使Cache获得尽可能高的命中率

高速缓存中的地址映像方法

  1. 直接映像(冲突大)
  2. 全相联映像(冲突小)
  3. 组相联映像(冲突较小)

中断

  1. 中断向量:把所有中断服务程序的入口地址（或称为中断向量）汇集为中断向量表
  2. 中断响应时间：发出中断请求开始，到进入中断服务程序
  3. 保存现场：返回执行源程序

输入输出（IO）控制方式

  1. 程序查询方式:
  2. cpu和I/O（外设）只能串行工作，CPU需要一直轮询检查，长期处于忙等状态。CPU利用率低
  3. 一次只能读/写一个字8bit
  4. 由CPU将数放入内存
  5. 中断驱动方式：
    1. I/O设备通过中断信号主动报告I/O操作已完成
    2. CPU和I/O（外设）可并行工作
    3. CPU利用率得到提升
    4. 一次只能读/写一个字8bit
    5. 由CPU将数据放入内存
  6. 直接存储器方式（DMA）
    1. CPU和I/O（外设）可并行工作
    2. 仅在传输数据块的开始和结束时才需要CPU的干预
    3. 有外设直接将数据放入内存
    4. 一次读写的单位为快而不是字
    5. DMA操作占用系统总线资源

总线Bus（了解）

数据总线，地址总线，控制总线

PCI总线，内总线，采用并行传输方式

SCSI:并行外总线

总线复用可以减少总线中信号线的数量

加密技术与认证技术

窃听 ：加密

篡改：摘要

假冒，否认：数字证书，数字签名

加密技术：

1. 对称加密（私有密钥加密）：加密和解密是同一把密钥，密钥分发有缺陷
        优点：加密解密速度快，适合大量明文数据
2. 非对称加密（公开密钥加密）：加密和解密不是同一把密钥，一共有两把密钥，分别是公钥和私钥
        公钥加密，验证
        私钥解密，签名
        不能通过一把推出另一把
        
        用接收方的公钥加密明文可以实现防止窃听的效果
        密钥分发没有缺陷
        1. 加密解密速度慢
3. 混合加密

认证技术：

认证处理主动攻击

  1. 摘要：将发送的明文进行Hash算法后得到摘要放在密文后一起发出，与接收方解密后的明文进行相同Hash算法得到的摘要进行对比，如果一致则没有篡改，否则有篡改

  2. 数字签名：发送方用自己的私钥对摘要进行签名（加密），得到数字签名放在密文后一起发送过去
    接收方用发送方的公钥对数字签名进行验证（解密）如果验证成功则该消息没有被假冒且不能否认，否则该消息的真实性为假冒发送

  3. 数字证书：

         用户向CA机构申请数字证书
     将个人信息和公钥发给CA机构
     CA机构颁发给用户数字证书
     数字证书用CA的私钥进行签名（加密）
     用CA的公钥验证（解密）数字证书
     得到用户的公钥
